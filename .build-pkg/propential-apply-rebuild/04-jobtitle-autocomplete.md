# Job title field — type-ahead autocomplete (Step 3)

Applies to `jobTitle` (data-fs-id 119531997) AND `prevJobTitle` (data-fs-id 119532001).

Build a custom **combobox / type-ahead**, NOT a plain `<select>`:
- A text input the user types into.
- As they type (after ~1 character), show a filtered dropdown of matching occupations
  (case-insensitive "contains" match against the 358-item list below).
- Results filter live as the user types; the user MUST pick an item from the filtered
  list. Free text that doesn't exactly match a list item is INVALID — block submit and
  show "Please select an occupation from the list".
- Selecting an item fills the input and sets the submitted value.
- Keyboard accessible: ArrowUp/ArrowDown to move, Enter to select, Esc to close.
  Use ARIA combobox roles: role="combobox", aria-expanded, aria-controls,
  role="listbox", role="option", aria-activedescendant.
- The submitted `value` must be the EXACT list string (so it maps to Formstack).
- Source the list from a single shared constant `JOB_TITLES` reused by both fields.
